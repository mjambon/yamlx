# Contributing to YAMLx

Thank you for your interest in YAMLx!

## How this project works

YAMLx is maintained by its author with the help of
[Claude Code](https://claude.ai/code). Because of this workflow,
**pull requests are generally not accepted** — the bottleneck is not
writing code but deciding what to build and verifying it is correct.
A patch that arrives as a PR requires the same review effort as one
written from scratch, without the benefits of having been shaped
interactively.

The most valuable contribution you can make is a **well-written issue**.

## Reporting bugs

Please open a [GitHub issue](https://github.com/username/yamlx/issues)
and include:

- A minimal YAML input that triggers the problem.
- The output you got and the output you expected.
- The YAMLx version (or commit hash).

## Proposing new features

Open a GitHub issue describing:

- The use case you are trying to solve.
- A proposed API or CLI interface, with examples.
- Any edge cases or trade-offs you have already thought through.

A detailed proposal that anticipates the hard questions is much more
likely to be accepted and implemented quickly. Vague requests may be
closed or left open indefinitely.

## What happens next

Issues that are accepted will be implemented by the maintainer (with
Claude Code). You are welcome to follow up in the issue thread if you
want to discuss the design further.
